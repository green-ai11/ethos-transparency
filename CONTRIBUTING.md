# Contributing

Thank you for interest in **Ethos-32B transparency** and [SustainAI Global](https://sustainai.global/). This repository is the **public documentation** home for methodology and values — not the private codebase where model weights and training pipelines are built.

---

## What you can contribute here

We welcome improvements to:

- Clarity and accuracy of markdown documentation
- Cross-links, typos, and accessibility of public-facing text
- Suggestions for additional **public-safe** transparency artifacts (with maintainers’ approval)
- Issues and discussions that help us explain HOW and WHY we build Ethos

Open a pull request or issue in [green-ai11/ethos-transparency](https://github.com/green-ai11/ethos-transparency) with a clear description of your proposed change.

---

## What belongs elsewhere

| Contribution type | Where it happens |
|-------------------|------------------|
| Model training code, weights, agent/backend | Private development repo — **by invitation only** |
| Raw or full training datasets | Future Hugging Face `ethos-training-data-v1` — not via drive-by PR here |
| API keys, configs with secrets, infra | Never in this repo — see [config/README.md](config/README.md) |

Do not submit fabricated training examples, private repo excerpts, or confidential material.

---

## Green Points and community participation

Community recognition and tier benefits (including early **chat** access for **Tier 3 (Green Steward) and above**) are managed through the **Green Points** program on [sustainai.global](https://sustainai.global/). Contributing to transparency docs does not automatically grant model access; access follows [USAGE.md](USAGE.md).

---

## Code of conduct (summary)

- Be constructive, accurate, and respectful
- Do not harass contributors or misrepresent SustainAI’s nonprofit status
- Do not use this repo to distribute malware, spam, or non-public proprietary content

Maintainers may close issues or PRs that violate these norms or mission fit.
